# Phishing IOC Extraction Tool

A CLI-based Python utility designed for Security Operations Center (SOC) analysts to automate the extraction of Indicators of Compromise (IOCs) from raw email files (`.eml`). The tool extracts the originating sender IP, HTTP/HTTPS URLs from the email body, and attachment metadata, then cross-references them with live threat intelligence feeds (AbuseIPDB and VirusTotal).

This project was built step-by-step for a portfolio demonstrate of SOC L1 automation capabilities.

---

## Features

- **MIME Email Parsing**: Uses Python's native `email` library to decompose multi-part email messages.
- **Originating IP Extraction**: Parses the SMTP transmission chain (`Received` headers) from oldest to newest to identify the first external public IP.
- **URL Scanning**: Extracts all unique hyperlinks from both plain-text and HTML email body parts.
- **Attachment Hashing**: Computes MD5 and SHA-256 cryptographic hashes of attachment payloads to facilitate hash-based threat intel lookups.
- **Threat Intelligence Integration**:
  - **AbuseIPDB API v2**: Inspects the sender's IP for historical abuse reports and calculates an abuse confidence score.
  - **VirusTotal API v3**: Checks extracted URLs and attachment hashes against over 70 security vendor reputation databases.
- **API Rate-Limit Handling**: Automatically manages VirusTotal free-tier rate limits (4 requests/minute) by applying a smart 15-second query cooldown.
- **Professional SOC CLI Output**: Uses `colorama` to color-code risk thresholds (Red = Malicious, Yellow = Suspicious, Green = Clean) and `tabulate` to present data in structured grids.
- **Structured JSON Export**: Supports exporting all raw indicators and threat intelligence scores to a clean JSON file for integration into ticket templates or SIEMs.

## Installation & Setup

### Prerequisites
- Python 3.9 or higher installed.

### 1. Clone the Repository
```bash
git clone https://github.com/yourusername/phishing-ioc-extractor.git
cd phishing-ioc-extractor
```

### 2. Set Up a Virtual Environment (Recommended)
```bash
# Create the virtual environment
python -m venv venv

# Activate it (Windows)
.\venv\Scripts\activate

# Activate it (Linux/macOS)
source venv/bin/activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Configure API Keys
Copy the template or edit the existing `.env` file in the root directory:

```ini
# Threat Intelligence API Keys
# Replace placeholders with your actual keys
ABUSEIPDB_API_KEY=YOUR_ABUSEIPDB_API_KEY_HERE
VIRUSTOTAL_API_KEY=YOUR_VIRUSTOTAL_API_KEY_HERE
```

*Note: You can register for free developer accounts on [AbuseIPDB](https://www.abuseipdb.com/) and [VirusTotal](https://www.virustotal.com/) to obtain these API keys.*

## Usage

Run the tool by passing the path of a raw `.eml` file as a positional argument.

### Basic Analysis
```bash
python analyzer.py samples/phishing_test.eml
```

### Export Report to JSON
To save the results to a file for SIEM ingest or ticket attachment, use the `-o` or `--output` flag:
```bash
python analyzer.py samples/phishing_test.eml -o report.json
```

## Key Technical Decisions Explainers (For Interviews)

If asked about this code in interviews, here are the key technical concepts implemented:

### Q1: Why parse the *oldest* (bottom-most) `Received` header for the sender IP?
In SMTP routing, every mail transfer agent (MTA) that handles the message prepends a new `Received` header at the top of the email. Hence, the top header is the last hop (e.g., your Google/Microsoft mail server), and the bottom-most header is the first hop (where the email entered the internet). We parse the headers from bottom-up, skipping local/private subnet IP ranges (e.g. `10.x.x.x` or `192.168.x.x`), to locate the first public IP address of the sender.

### Q2: How are URLs queried on VirusTotal API v3?
VirusTotal v3 requires a URL identifier, which is generated by base64-encoding the URL in a URL-safe format and removing any trailing `=` padding:
```python
url_id = base64.urlsafe_b64encode(target_url.encode()).decode().strip("=")
```
This URL-safe identifier is then appended directly to the VT endpoint path: `https://www.virustotal.com/api/v3/urls/{url_id}`.

### Q3: How is VirusTotal's 4 req/min free tier rate limit handled?
We implement a global rate-limiting cooldown system. We record the timestamp of the last request (`last_vt_request_time`). Before launching a new query, we calculate the elapsed time. If less than 15 seconds have passed, the script sleeps (`time.sleep()`) for the difference, ensuring we never hit `HTTP 429 Too Many Requests`.
